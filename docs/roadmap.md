# План разработки версии 1.0

Этапы описывают порядок реализации одного полноценного production-продукта. Публичный запуск происходит только после завершения всех этапов и выполнения критериев качества.

## Этап 0 — проектирование (3–5 рабочих дней)

- Утвердить решения из `architecture.md`.
- Перевести текущий `index.html` в карту экранов и UI-токены; согласовать mobile-макеты.
- Описать OpenAPI-контракт, правила доступа, модель угроз и тексты privacy/community guidelines.
- Поднять локальный Docker Compose: Postgres, Redis, MinIO, Mailpit, LiveKit; подготовить эквивалентные staging и production-конфигурации.

Результат: репозиторий, CI, окружения и спецификация, по которой можно разрабатывать без переделок.

## Этап 1 — безопасный фундамент (5–8 дней)

- Next.js-каркас, дизайн-система и адаптивная оболочка личного кабинета.
- NestJS: user, вход, подтверждение, сессии, роли, consent.
- Профили, интересы, языки, privacy settings; перенос визуального прототипа в реальные компоненты.
- Защищённый раздел администратора: пользователи, профили, роли и журнал аудита.

Критерий: пользователь регистрируется, безопасно входит, редактирует профиль с телефона и desktop.

## Этап 2 — знакомства и сообщения (7–10 дней)

- Правила рекомендаций, скрытие кандидата, блокировка.
- Личные диалоги, группы до 10, история, read cursors, WebSocket, reconnect.
- Жалобы на аккаунт/сообщение, очередь модерации, санкции.
- E-mail/in-app-уведомления, лимиты и антиспам первой версии.

Критерий: два пользователя могут найти друг друга, общаться в реальном времени, заблокировать/пожаловаться; модератор обрабатывает случай с аудитом.

## Этап 3 — звонки (5–8 дней)

- Self-hosted LiveKit и coturn в staging.
- Создание и приглашение в комнаты, token endpoint, роли, лимит 10, выход/завершение комнаты.
- Проверка камеры/микрофона, экран ожидания, состояния сети и fallback-аудио.
- Нагрузочный и мобильный тест; мониторинг сетевых метрик.

Критерий: приглашённые участники стабильно подключаются к аудио/видео на desktop и mobile browser, лишний пользователь не проходит в закрытую комнату.

## Этап 4 — production readiness (3–5 дней)

- Staging → production, TLS, домены, резервное копирование, алерты, Sentry.
- Security review: права API, rate limit, секреты, webhook signature, CSP/CORS/CSRF.
- E2E сценарии, нагрузочный smoke-test, runbook администратора и процесс отката.

Критерий: восстановление из бэкапа проверено, мониторинг работает, выпуск воспроизводим одним pipeline.

## Оценка и объём версии 1.0

Полный объём выше — ориентировочно **23–36 рабочих дней** для одного опытного full-stack-разработчика без ожиданий по дизайну/контенту и без платёжной интеграции. Обещать полноценную версию 1.0 за 10–20 дней было бы нереалистично: в неё входят права, модерация, realtime, видеоинфраструктура, тесты и production-hardening.

Версия 1.0 включает все четыре этапа: профили, рекомендации, личные и групповые чаты, модерацию, аудио/видео, адаптивный UI, мониторинг и production-подготовку. Сначала не строить отдельные продукты: ML-рекомендации, запись звонков, нативные приложения и платежи. Простая объяснимая выдача анкет полезнее и безопаснее на старте; при этом контур для будущих платных функций уже заложен в архитектуре.
